Unix MCP

NixOS & home-manager flake operations — 36 tools across 7 modules

Shell (3 tools)

MCPshell_exec

Execute a shell command and return stdout/stderr. Runs in the user's default shell with PATH configured for nix-profile.

string — Shell command to execute
string? — Working directory (default: $HOME)
number? — Timeout in ms (default: 30000)
MCPshell_read_file

Read a file from the local filesystem. Supports text files and returns content with line numbers.

string — Absolute file path
number? — Max lines to read (default: all)
MCPshell_write_file

Write content to a file. Creates parent directories if needed.

string — Absolute file path
string — File content

Nix (6 tools)

MCPnix_build

Run nix build in a flake directory. Returns build output path.

string — Path to flake directory
string? — Build target (default: default package)
MCPnix_switch

Apply a NixOS or home-manager configuration via build.sh switch.

string — Path to flake directory
string? — Configuration profile name
MCPnix_search

Search nixpkgs for a package by name. Returns matching packages with version and description.

string — Package name to search
string? — Nixpkgs channel (default: nixos-24.11)
MCPnix_info

Get detailed info about a nix package — version, dependencies, description, license.

string — Package attribute path (e.g. nixpkgs#ripgrep)
MCPnix_flake_metadata

Get metadata for a flake — inputs, locks, last modified dates.

string — Path to flake directory
MCPnix_flake_update

Update flake inputs (nixpkgs, home-manager, etc.) via nix flake update.

string — Path to flake directory
string? — Specific input to update (default: all)

Git (4 tools)

MCPgit_status

Show working tree status for a repository. Returns modified, staged, and untracked files.

string — Repository path
MCPgit_log

Show recent commit history with hash, author, date, and message.

string — Repository path
number? — Number of commits (default: 10)
MCPgit_diff

Show diff of working tree or between commits.

string — Repository path
string? — Commit ref or range (default: HEAD)
MCPgit_commit

Stage files and create a commit.

string — Repository path
string — Commit message
string[]? — Files to stage (default: all modified)

System (5 tools)

MCPsystem_info

Get system information — OS, kernel, arch, hostname, uptime, memory, disk usage.

MCPsystem_processes

List running processes with PID, CPU%, MEM%, and command. Sorted by resource usage.

string? — Sort by: cpu, mem, pid (default: cpu)
number? — Max processes to return
MCPsystem_services

List systemd services with status (running, stopped, failed).

string? — Filter by name pattern
MCPsystem_journal

Read systemd journal logs for a unit or the whole system.

string? — Systemd unit name
number? — Number of recent lines (default: 50)
string? — Min priority: emerg, alert, crit, err, warning, notice, info, debug
MCPsystem_network

Show network interfaces, IP addresses, routes, and listening ports.

Scripts (8 tools)

MCPscript_list

List available custom scripts from nix-profile bin directory.

MCPscript_run

Execute a named script with arguments.

string — Script name
string[]? — Arguments to pass
MCPscript_create

Create a new shell script in the flake's script directory.

string — Script name
string — Script content
MCPscript_edit

Edit an existing script's content.

string — Script name
string — New script content
MCPscript_delete

Remove a script from the flake.

string — Script name
MCPscript_show

Display a script's source code.

string — Script name
MCPscript_search

Search scripts by name or content pattern.

string — Search pattern
MCPscript_deps

Analyze script dependencies — which commands/packages a script requires.

string — Script name

Drift (3 tools)

MCPdrift_check

Check version drift across all tracked packages and flake inputs. Compares current vs latest versions.

string? — Filter by package name
boolean? — Show only outdated entries
MCPdrift_plan

Generate an update plan for outdated packages. Computes new hashes, shows diffs.

string? — Specific package to plan (default: all outdated)
boolean? — Show plan without applying (default: true)
MCPdrift_apply

Apply a drift update plan — update version strings, hashes, and flake inputs in source files.

string? — Plan ID from drift_plan (default: latest)
