# Strategic Map Viewer

This project allows you to visualize strategic data on interactive Highcharts maps, configured via CSV files.

## Files

-   `index.html`: The main web page that displays the map.
-   `mymaps.py`: A Python script that reads configuration and data from CSV files, processes it, and generates `mymaps.js`.
-   `input_data.csv`: Contains the strategic data for geographic regions (countries, states, etc.).
-   `config.csv`: Configures the map type, joining strategy, and data column mappings.
-   `mymaps.js`: Generated by `mymaps.py`, this file contains the processed data and map configuration used by `index.html`.

## Setup and Usage

1.  **Configure your map (`config.csv`)**:
    Open `config.csv` and define your map settings. See the [Configuration Parameters](#configuration-parameters) section for details on available options.

2.  **Prepare your data (`input_data.csv`)**:
    Open `input_data.csv` and enter your strategic data. Ensure the column headers match the `data_X_column` settings in `config.csv`.

3.  **Generate Map Data (`mymaps.py`)**:
    Run the Python script to process your CSVs and generate the JavaScript file for the map:
    ```bash
    python3 mymaps.py
    ```
    This will create or update `mymaps.js`.

4.  **View the Map (`index.html`)**:
    Open `index.html` in your web browser. The map should render according to your `config.csv` and `input_data.csv` settings.

## Configuration Parameters

The `config.csv` file uses a `setting,value` format. Here are the available parameters:

---

#### 1. `map_path`
-   **Description**: Specifies the path to the Highcharts map data file to be loaded. This determines which geographic area (e.g., world, specific country, region) is displayed.
-   **Options**:
    -   **Highcharts Map Collection**: Highcharts provides a vast collection of map data files. You can find these at `https://code.highcharts.com/mapdata/`.
    -   **Format**: The `value` should be the path *relative* to the `mapdata` directory.
    -   **Examples**:
        -   `custom/world`: The world map (default).
        -   `countries/us/us-all`: All states of the United States.
        -   `countries/de/de-all`: All states of Germany.
        -   `countries/gb/gb-all`: Counties of Great Britain.
        -   To find others, browse `https://code.highcharts.com/mapdata/` and take the path after `mapdata/` and before `.js`. For example, `countries/ca/ca-all.js` would be `countries/ca/ca-all`.
-   **Impact**: Directly controls the base map rendered.

---

#### 2. `join_by`
-   **Description**: Defines the property in the Highcharts map data (GeoJSON features) that will be used to match with the `code` value from your `input_data.csv`. This is crucial for correctly coloring and linking data to map regions.
-   **Options**:
    -   **`hc-key`**: (Recommended) This is the most reliable and commonly used identifier in Highcharts maps. It's typically a 2-letter ISO country code in **lowercase** (e.g., `us`, `ru`, `de`). Your `input_data.csv` `code` column should match this format.
    -   **`iso-a2`**: The standard 2-letter ISO country code, typically in **uppercase** (e.g., `US`, `RU`, `DE`). Use this if your `input_data.csv` `code` column contains uppercase ISO-A2 codes.
    -   **`iso-a3`**: The standard 3-letter ISO country code, typically in **uppercase** (e.g., `USA`, `RUS`, `DEU`). Use this if your `input_data.csv` `code` column contains uppercase ISO-A3 codes.
    -   Other less common properties might exist in specific map files (e.g., `fips`, `postal-code` for subregions). You would need to inspect the `.js` map data file to confirm.
-   **Impact**: Determines how your data points are linked to the map's regions. A mismatch will result in uncolored or unlinked regions.

---

#### 3. `title`
-   **Description**: The main title displayed at the top of your HTML page and potentially in the map chart itself.
-   **Options**: Any string value.
-   **Impact**: Changes the heading on your web page.

---

#### 4. `data_code_column`
-   **Description**: Specifies the name of the column in `input_data.csv` that contains the unique identifier for each geographic region (country, state, etc.). This column's values are matched against the `join_by` property of the map.
-   **Options**: Any column header string present in your `input_data.csv`.
-   **Impact**: Essential for data-to-map region linking.

---

#### 5. `data_name_column`
-   **Description**: Specifies the name of the column in `input_data.csv` that contains the display name for each geographic region. This name is typically shown in tooltips when hovering over a region.
-   **Options**: Any column header string present in your `input_data.csv`.
-   **Impact**: Provides user-friendly names for map regions.

---

#### 6. `data_role_column`
-   **Description**: Specifies the name of the column in `input_data.csv` that contains a descriptive "role" or status for each geographic region. This information is typically displayed in the tooltip.
-   **Options**: Any column header string present in your `input_data.csv`.
-   **Impact**: Adds contextual information to map tooltips.

---

#### 7. `data_sphere_column`
-   **Description**: Specifies the name of the column in `input_data.csv` that contains a category or "sphere" string (e.g., `RED_CORE`, `BLUE_ALLY`). These strings are used to look up the corresponding hex color code from the `COLORS` dictionary in `mymaps.py`.
-   **Options**: Any column header string present in your `input_data.csv`. The values in this column must match keys defined in the `COLORS` dictionary in `mymaps.py`.
-   **Impact**: Determines the color of each geographic region on the map. Unmatched sphere keys will default to `NEUTRAL_WEAK` color.
